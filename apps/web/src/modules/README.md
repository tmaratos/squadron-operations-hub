# Feature modules

Each module should use a vertical-slice structure where practical:

- `components` - UI
- `domain` - types and business rules
- `services` - application services
- `repositories` - data-access abstractions
- `actions` - server actions or route handlers
- `schemas` - module-specific validation
- `tests` - module tests
